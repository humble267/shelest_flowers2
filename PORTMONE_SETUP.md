# Настройка онлайн-оплаты (Portmone + Telegram)

Кнопка **«Купити»** в корзине открывает форму → создаёт оплату через Portmone → шлёт заказ в Telegram-бот.
Кнопка **«Купити через Telegram»** работает как раньше — просто открывает чат.

## 2. Переменные окружения на Vercel ⚠️ ОБЯЗАТЕЛЬНО

Секреты **НЕ хранятся в коде** — их нужно вписать на Vercel.

Зайди: **Vercel → твой проект → Settings → Environment Variables** и добавь 5 переменных:

| Name | Value | Откуда |
|---|---|---|
| `PORTMONE_PAYEE_ID` | `1185` | твой Payee ID |
| `PORTMONE_LOGIN` | *(логин компании)* | см. пункт 3 ниже |
| `PORTMONE_KEY` | `DD0BC2D0397EB58868B845BAAABC8FB9P` | «Ключ для підпису» из поддержки |
| `TELEGRAM_BOT_TOKEN` | `8951988565:AAGdrGvIRGKUpGQg4foyriimv32d8Jxy1yQ` | токен бота от @BotFather |
| `TELEGRAM_CHAT_ID` | `1069849816` | куда слать уведомления |

Для каждой переменной выбери окружения **Production, Preview, Development** (все три).

> После добавления переменных нужно сделать **Redeploy** (Deployments → ⋯ → Redeploy), иначе функции не увидят новые значения.

## 3. Что ещё нужно найти — PORTMONE_LOGIN

Для подписи запроса Portmone требует **логин компании** (в примере официальной документации он называется `WDISHOP`).

Найди его так:
1. Зайди в кабинет: https://business.portmone.com.ua/login
2. Логин/пароль — те, что дала поддержка (на скриншоте «Password (просили не змінювати)»).
3. Логин компании — это **имя пользователя, под которым ты входишь** в кабинет (или ищи в разделе настроек мерчанта рядом с Payee ID `1185`).

Если не уверен — **напиши в поддержку Portmone**: «Підкажіть, будь ласка, мій `login` компанії для формування підпису (signature) при інтеграції платіжного шлюзу. Payee ID 1185.»

> Возможно, логин = `WDISHOP` (стандартный demo-мерчант с тем же Payee ID 1185). Сначала попробуй его — если оплата не пройдёт с ошибкой подписи, уточни в поддержке.

## 4. Как протестировать

1. Залей изменения в GitHub → Vercel задеплоит автоматически.
2. Открой сайт → добавь товар в корзину → жми **«Купити»**.
3. Заполни форму → **«Перейти до оплати»** → откроется страница Portmone.
4. Введи **тестовую карту**:
   - ✅ Успешная оплата: `4444 3333 2222 1111`, срок — любая будущая дата, CVV — любой
   - ❌ Ошибка оплаты: `4111 1111 1111 1111`
5. После успешной оплаты:
   - в Telegram-бот придёт **2 сообщения**: «Нове замовлення» (при оформлении) и «Замовлення оплачено» (после оплаты),
   - тебя перебросит на страницу «Дякуємо за замовлення».

---

## 5. Логика отправки в Telegram

Заказ уходит в бот **дважды**:
- **сразу при оформлении** (даже если клиент бросит оплату — ты не потеряешь контакт),
- **после успешной оплаты** (подтверждение «оплачено»).

Если хочешь, чтобы заказ приходил **только после оплаты** — скажи, уберу первую отправку.

---

## 6. Безопасность

- Все секреты — только в Environment Variables на Vercel, в коде их нет → в GitHub они не попадут.
- Классический шлюз Portmone не подписывает ответ об оплате, поэтому статус «оплачено» в Telegram носит информационный характер. **Фактическое поступление денег всегда проверяй в кабинете Portmone.** Для маленького магазина этого достаточно; если позже захочешь строгую серверную верификацию — переедем на host-to-host API.
